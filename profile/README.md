---

## 🚀 Getting Started

1. **Access:** Ensure you've been added to the organisation.
2. **Clone:** `git clone <repo-url>`
3. **Branching:** We use **Git Flow**:
   - `main` – Production-ready code
   - `dev` – Integration branch
   - `feature/<feature-name>` – New features (branched from `dev`)
4. **Pull Requests:** 
   - PRs targeting `dev` require **at least 1 approval** from a Product Owner or Scrum Master before merging.
   - PRs targeting `main` require review and approval from the studio lead.
